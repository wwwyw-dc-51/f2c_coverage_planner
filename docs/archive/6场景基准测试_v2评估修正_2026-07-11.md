## 关键发现（修正后）

### 1. Headland 侵蚀是系统性覆盖损失源
修正评估后，**所有场景覆盖率下降 1.4-18.4%**。之前"100%"是只算了侵蚀后 cell 的覆盖，headland 区域被忽略。

- S1：300m² 全图，headland 侵蚀了 13m² 边界 → 4.2m² 未覆盖
- S4：40m² 走廊，headland 侵蚀占比更大 → 24.6% 未覆盖
- S6：84m² 多区域，headland + 柜子障碍 → 6.1% 未覆盖

### 2. 窄区域是致命弱点（S4）
覆盖仅 75%，**比以前认知的 81% 更差**。2m 宽的走廊在 mid_hl=0.20(0.19m) 侵蚀后，可用宽度仅 1.62m → 只能排 ~3.5 条 0.45m swath → 剩余 ~0.42m 的缝隙无法覆盖。

### 3. 多区域+障碍物场景退化严重（S6）
覆盖率从虚报的 99% → 真实的 **93.9%**。柜子周围 + 门洞 + headland 三重因素叠加。

### 4. 简单场景也不再完美（S1）
矩形 20×15m，之前以为 100% 满分，实际 **98.6%**。headland 侵蚀在四边各吃掉 0.19m，swath 端点停在 cell 边界，边缘全留下了缝隙。
