## 1. 评分体系 v8

### 设计理由

实际运行环境下，受限于机器人物理尺寸 (0.95m)、固定宽度 swath (0.45m) 无法完美整除区域尺寸、
边界掉头空间等因素，100% 覆盖率几乎不可能达成。旧公式 `gate = coverage^10` 在 95% 时仍给 0.60 分，
无法有效区分"优秀"(99%+) 和"不合格"(90%)。

### 公式

```
CoverageGate = ((coverage - 0.90) / 0.09)³   钳制到 [0, 1]
综合得分 = CoverageGate × efficiency_score
```

### 门控曲线

| 覆盖率 | gate | 评价 |
|:------:|:----:|:----:|
| ≥99% | 1.00 | 满分段 |
| 98% | 0.70 | 高分段 |
| 97% | 0.47 | 中段 |
| 95% | 0.17 | 低分段 |
| 93% | 0.04 | 临界 |
| ≤90% | 0.00 | 不合格 |

### 架构

CoverageGate 是**乘性门控**（gate × efficiency），不是加法加权。覆盖率不达标时得分直接归零，
覆盖率达标后由效率维度（工作比、重叠率、转弯、平滑度、可行性）决定分数。

## 2. 边界补线 v6→v7

### v6 问题
点级检测（距离 < half_w + 跨度 > 80%）在窄走廊被转弯路径误判。

### v7 段级检测
改为路径段级别：扫描相邻点对（段），仅统计与 swath 同向（夹角<30°）且在覆盖范围内的段。
找最长连续同向段序列判断覆盖。转弯段自动被过滤。

### S4 窄走廊分析
v7 正确识别所有 6 条 swath 均被相邻覆盖。90% 覆盖率根因：固定宽度 swath (0.45m)
在 2m 宽走廊中，最左 swath 中心在 x=0.41，覆盖范围 [0.185, 0.635]，左墙 [0, 0.185] 无覆盖。
需 swath 边距对齐优化（动态重叠率），非补线问题。

## 3. Cell 连接优化

### 问题
多 cell 场景（如 notched）中，cell 间连接不考虑起止点位置，产生不必要的长距离斜线。

### 修复
Boustrophedon 排序后，对每个 cell 测试反转 swath 顺序是否能缩短与前一 cell 的连接距离。
反转至少缩短 30% 且不穿越孔洞时才执行。

### 孔洞穿越检测
反转优化加入孔洞穿越检测：采样连接线段，用射线法检测是否经过孔洞内部。若穿越则跳过该 cell 的反转。

### 效果 (notched)

| 指标 | 修复前 | 修复后 |
|------|:------:|:------:|
| 覆盖率 | 97.32% | **98.60%** |
| 得分 | 44.0 | **70.0** |
| 长斜线 | 3 条 | **0 条** |
| 孔洞穿越 | 3 条 | **0 条** |
