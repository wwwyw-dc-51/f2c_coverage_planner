## 一、Wageningen 硕士论文（2024）

**标题**：Workflows for coverage path planning for complex agricultural fields
**作者**：Bas Peijnenburg，导师 Gonzalo Mier（F2C 作者）
**数据**：AAN-Brabant 数据库，荷兰北布拉班特省农地，每类 1000 块田

### 三种工作流对比

| | W1（无分解） | W2（Boustrophedon/梯形） | W3（边缘延伸分解） |
|---|---|---|---|
| 小凹田覆盖率 | **78.4%** | 71.7% / — | — |
| 大凹田覆盖率 | **93.7%** | 88.1% / 71.4% | 93.0~93.3% |
| 障碍田覆盖率 | **79.7%** | 63.9% / 45.5% | 77.6~78.0% |
| 专家可行性评分 | 3~4 名 | 5~8 名 | **1~2 名（最优）** |

### 关键发现

1. **梯形分解最差** — 子区域太多，覆盖率掉到 45%。跟我们担心的"分解碎片化"一致
2. **Boustrophedon 中等** — 覆盖率还行但不如不分解
3. **W3（边缘延伸）最均衡** — 覆盖率接近 W1，swath 更少，专家评为最实用
4. **Alpha 权重**（覆盖率 vs swath 数的平衡因子）— 低 alpha 梯形赢，高 alpha W1 赢，中 alpha W3 赢。我们的 CoverageGate ^10 = 极高 alpha（几乎只关心覆盖率）
5. **凸度比（CR）影响** — 越凹 W3 越好，越凸 W1 越好。我们场景外边界规整 = 高 CR = W1 就够

### 对我们的启示

- 我们当前架构 = W1 + 优化（多角度 + 过滤 + RDP + Reeds-Shepp），方向正确
- 梯形分解慎用——碎片化严重，需要合并逻辑
- 自适应 headland 符合专家建议——不必要的 headland 应该移除
- 未来可考虑 W3 的边缘延伸分解（对大凹田有明显优势）

## 二、F2C vs 智元对比文档

| # | F2C 问题 | 我们的对策 |
|---|---------|-----------|
| 1 | 框大于地图时无路径 | `boundary_type` 边界策略 |
| 2 | 长廊区域漏覆盖 | 自适应 headland + 窄通道诊断 |
| 3 | 路径显示与实际不一致 | `path_simplify_turn_threshold` |
| 4 | 不规则凸起多目标点 | 分解角度优化 + 子区域过滤 |

**智元的问题**：会在不可通行区域规划路径并卡住 → 我们有掉头可行性检查 + RViz 标记

## 三、语雀文档（2026-07-08 已获取）

### 工业清洁场景特点
1. 室内平整地面，无需考虑起伏
2. 清洁头宽度精准、转弯受限于墙角/障碍
3. 重叠低、覆盖率高（不漏扫）
4. 避免狭小区域频繁掉头 → 我们加了 Reeds-Shepp + 自适应 headland
5. 节能优先、避免不必要转向

### 推荐参数 vs 我们
| 参数 | 语雀 | 我们 | 差异 |
|------|------|------|------|
| robot_width | 0.6m | 0.95m | |
| min_radius | 0.5m | 0.1m | 我们是差速驱动 |
| headland | 0.1m | 0.25+0.0 | |
| swath_spacing | 0.58m（3%重叠） | 0.45m（0重叠） | **缺重叠参数** |

### 我们缺失的功能
1. ❌ swath 重叠参数（防止定位误差漏缝）
2. ❌ 地图预处理（形态学腐蚀去小缝隙）
3. ❌ 转弯段速度控制（曲率限速 / Nav2 velocity smoother）

### Nav2 Coverage Server 参考
Fields2Cover 在 ROS2 生态中有标准 Coverage Server 实现，参数体系可作为参考：
- `robot_width` / `operation_width` / `min_turning_radius`
- `linear_curv_change` — 曲率变化率（我们没有）
- `default_allow_overlap` — 边缘允许重叠（我们没有）
- `default_swath_type` — LENGTH / COVERAGE / NUMBER 评分策略
