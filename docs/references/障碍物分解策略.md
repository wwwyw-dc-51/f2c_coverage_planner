# 障碍物场景下的最优分解策略

> 收集日期：2026-07-07
> 目的：了解有障碍时如何选择最优的分解策略和角度

---

## 为什么 Boustrophedon 够用

### 与 Trapezoidal 分解对比

| | Trapezoidal | Boustrophedon |
|---|---|---|
| 触发条件 | 每个障碍顶点都切一刀 | 只在障碍"出现/消失"时切 |
| Cell 数量 | 多（过度切分） | 少 |
| 冗余移动 | 高 | 低 |
| 复杂度 | 低 | 中 |

```
同一场景对比：
Trapezoidal:  切出 8 个 cell，每个独立走，多 4 次掉头
Boustrophedon: 切出 4 个 cell，相邻自然合并
```

**结论**：Boustrophedon 已经是更优的选择，Fields2Cover 默认就用的它。

---

## 分解角度 vs Swath 角度

这是两个不同的优化维度：

| | 分解角度 | Swath 角度 |
|---|---|---|
| 用途 | 决定怎么切 cell | 决定每个 cell 里 swath 方向 |
| 优化目标 | cell 数量最少 + 总面积最小 | swath 条数最少 |
| 我们已在做 | ❌ 默认 0° | ✅ 边缘方向剪枝 |

**关键发现**：最优分解方向也一定是**垂直于多边形某条边**的方向（Huang, 2001 ICRA）。

---

## MSA（Minimum Sum of Altitudes）方法

定理（Huang 2001）：对任意多边形，最优的 sweep 方向（最小化各 cell 高度之和）出现在**垂直于某条边**的方向。

### 算法流程
1. 提取所有边方向，计算垂直方向作为候选分解角
2. 对每个候选角运行 Boustrophedon 分解
3. 计算 "swath 条数之和"（Sum of Altitudes）
4. 选最小的那个

### 能否与现有 Swath 角度优化合并？

可以。分解角度的候选和 swath 角度的候选来自**同一组边缘方向**：
- 分解角度 = 边缘垂直方向（atan2(dx,dy) + 90°）
- Swath 角度 = 边缘平行方向（atan2(dy,dx)）

两者只差 90°，共用同一组边缘信息。

## 实际建议

对于嵌入式系统，分解角度可以离线预处理：

```
预处理（离线，做一次存下来）：
  1. 提取所有多边形边缘方向
  2. 生成候选分解角 = 边缘垂直方向
  3. 对每个候选角：Boustrophedon 分解 → 计算总 swath 条数
  4. 选最优分解角 + 对应的 cell 分割结果
  5. 存入配置文件

运行时（在线，快）：
  1. 读配置文件 → 已知最优分解角
  2. 直接分解 → 每个 cell 走 swath 角度优化（边缘方向剪枝）
  3. 输出路径
```
