## 1、Fields2Cover在工业机器人的算法设置
## （1）工业清洁场景特点
1. **环境为室内或平整地面**，无需考虑地面起伏；
2. **清洁头宽度精准、转弯空间较大但受限于墙角或障碍物**；
3. **清洁重叠要低，覆盖率要高（不漏扫）**；
4. **需避免在狭小区域中频繁掉头**；
5. **节能优先，避免不必要的转向和长路径**。



## （2）推荐参数配置
+ f2c.robot.width = 0.6;                       // 清洁宽度 60cm
+ f2c.robot.min_radius = 0.5;                  // 差速底盘的转弯半径



+ f2c.headland_dist = 0.1;                     // 贴边清洁缓冲区



+ f2c.opt_dir_type = OptDir::MIN_TURNS;        // 减少转弯次数
+ f2c.order_type = PathOrder::Boustrophedon;   // 牛耕式清洁方式
+ f2c.turning_type = TurnType::DubinsCurves;   // 合理转弯半径模型



+ f2c.swath_spacing = 0.58;                    // 轻微重叠，避免漏扫
+ f2c.coverage_overlap = 0.03;

## （3）额外建议
1. **地图简化处理：**室内地图中包含很多小缝隙或拐角，应使用形态学腐蚀（erosion）预处理，以避免生成不可清扫区域的路径。
2. **路径速度控制（结合你前面的问题）：**在转弯段配合曲率限制速度控制，或者使用 `nav2_velocity_smoother`。
3. **动态重规划支持（增量清洁）：**如果你清扫的是开放空间，如仓库，可以配合动态地图更新，实现局部区域重清扫。

## 3、F2C算法如何进行区域分割
（1）F2C算法解决问题的核心思想

**Fields2Cover**（简称 F2C）是一个为农业机器人和自主移动机器人开发的**全场地覆盖路径规划算法框架**。  
它的核心思想是：

**将复杂、不规则场地分解成规则子区 → 为每个子区生成规则的作业行（Swaths） → 用平滑曲线连接这些行 → 形成连续、覆盖率高的路径。**

技术上，它融合了几种成熟的计算几何与路径规划算法：

+ **多边形分解算法（Polygon Decomposition）**
+ **作业行生成与评分算法（Swath Generation & Evaluation）**
+ **路径连接优化（Path Connection with Dubins / Reeds–Shepp curves）**
+ **边界剥离与规整化算法（Headland Generation）**

它不是一个单一算法，而是一套**模块化规划体系**。  
这些模块组合起来，就能解决你提到的“坑坑洼洼边界 → 生成规整路径”的问题。

（3）关键技术原理

### 1️⃣ 场地分解（Field Decomposition）
#### 📘 原理：对输入的多边形场地（可能有凹陷、洞、复杂边缘），F2C 先进行**几何分解**。  
常用两种算法：
+ **Trapezoidal decomposition（梯形分解）**：把不规则多边形拆成多个规则梯形；
+ **Boustrophedon decomposition（往返式分解）**：基于拓扑连通性拆分场地，保证每个子区可一次往返完成。

#### 🧩 效果
+ **<font style="color:#AD1A2B;">把边缘的凹坑“割掉”，转为多个简单子区；</font>**
+ 每个子区的边界更加平滑，适合生成平行作业行；
+ 从全局看，坑洼边界不会直接干扰主路径的走向。



### 2️⃣ 头地生成（Headland Generation）
#### 📘 原理：头地（Headland）是农业作业中“离地边界一定距离”的区域。  
在 F2C 中，通过对场地多边形做**内缩（inset / offset）运算**实现：
用一个偏移宽度（headland width），向内平移场地边界，生成一个更规整的“作业区域”。

#### 🧩 效果
+ 自动去掉边缘的坑洼、凸角；
+ 生成一个几何上更规则的“内层区域”；
+ 减少路径在边界不连续处震荡。



### 3️⃣ 作业行生成（Swath Generation）
#### 📘 原理：在生成的规则内区内，F2C 使用一种**角度扫描 + 评分优化算法**寻找最优作业方向。
算法步骤如下：

1. 遍历 0°~180° 的多个角度；
2. 在每个角度上生成一组平行线；
3. 计算每组线与场地的交集；
4. 根据评分函数选出最佳角度（最短路径 / 最少行数 / 最大覆盖率）。

#### 📊 评分函数举例
+ **Length**：总路径长度最短；
+ **Coverage**：覆盖率最高；
+ **Number**：行数最少。



### 4️⃣ 路径连接与平滑化（Path Planning）
#### 📘 原理
在作业行之间，F2C 使用**Dubins 曲线**或**Reeds–Shepp 曲线**生成光滑转弯路径。

这些曲线的特点：

+ 满足车辆的最小转弯半径约束；
+ 曲率连续，转向自然；
+ 生成的路径段可直接发送给差速 / Ackermann 控制器。、
