# 2026-07-10 工作日志

## 核心改动：cell 级遍历 + 孔洞感知路由

### 架构变更

```
旧管线: swaths → genRoute(TSP全局排序) → 方向B bbox重排 → 路径
新管线: swaths → cell圆形排序 → genRoute(headland路由) → 路径
```

**核心理念**：先定 cell 遍历顺序（绕孔洞圆形），再在 cell 内部铺 swath。TSP 优化限制在 cell 内，cell 间连接走 headland 路径避免越界。

### 具体改动

#### 1. cell 圆形遍历排序（替代半平面分组）
- 计算每个 cell 的 swaths 质心
- 按孔洞质心极角排序（顺时针）
- 保持 cell 分组结构传给 genRoute

#### 2. 边界间隙填补 `fillBoundaryGaps()`
- 新增函数，检测每个 cell 最外层 swath 到边界的距离
- 间隙 > 0.02m → 在边界补一条 swath
- Closed 硬边界：补的 swath 端点自动内缩 0.03m
- 使用 `getLinesInside` 裁剪到 cell 内

#### 3. 全局优化缓存
- 最优角度的 swaths 在测试阶段缓存（`best_cell_swaths`）
- 避免二次 `generateSwaths` 调用

#### 4. 死代码清理
- `#if 0` 包裹后减法代码块（永远 0 affected）
- 移除方向B bbox 重排逻辑（~100行，被 cell 级排序取代）
- 移除半平面分组逻辑

#### 5. UnionOp MultiPolygon 修复
- Cell 合并时 `result.size() > 1` 的多余 geometry 作为独立 cell 加入

#### 6. 计时修正
- `planning_end` 从评估块移到路径发布之后、孔洞检测之前
- 纯规划时间不含诊断/评估

#### 7. 评估切网格法
- `eval_use_grid_method:=true` + `eval_grid_resolution:=0.05`
- 精确覆盖率计算替代几何估算法

### 尝试过但回退的方案

| 方案 | 结果 | 原因 |
|------|------|------|
| 方向B bbox 重排 | 34次交换，但路径变长+穿越增加 | bbox 误判太多，打乱 TSP 顺序 |
| 策略3 跨组整组交换 | 路径从 692m→818m | 破坏 TSP 优化 |
| 4组侧边分组 | 必有1条对角线穿越 | 4组环的哈密顿路径几何必然 |
| 2组半平面分割 | 仍有穿越 | TSP 在组内仍可跨 cell 连接 |
| 手动 route 构建(Boustrophedon) | 越界 | 直接连线不走 headland |

## 下午：边界填补探索（大部分回退，最终回归干净版）

### 尝试过的边界填补方案

| 方案 | 结果 | 原因 |
|------|------|------|
| `fillBoundaryGaps` 投影法 + `allow_overlap=false` | 越界+路径变长 | 关掉 allow_overlap 后 F2C swath 布局变了，genRoute 被搞乱 |
| `swath_endpoint_shrink_distance:=0.0` | 覆盖率 93% | closed 保底值 0.3m 生效，swath 大量被切 |
| 投影法 + `allow_overlap=true` | 浮点精度误触发填充 | `start=0.000` 实为 1e-15 |
| 阈值降到 0.02m | 仍有误触发 | 同上 |
