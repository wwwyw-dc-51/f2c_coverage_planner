# 2026-07-20 工作日志

## v9.11 — S7 穿洞修复 + per-cell 离群 veto 完善

### 背景

S7（工厂车间 30m×20m，10 孔洞）在 v9.9 引入 per-cell 离群 veto（方向 B）后，出现路径穿越孔洞/柱子的严重问题。老板的稳定妥协版没有此问题，要求精确定位根因而非过度工程化。

### 根因定位

**`greedyCellOrder` 中穿洞连接仅使用软惩罚（+1000）**，而非硬约束。

`path_planner.cpp:610-614` 的 `connectionCost` lambda：
```cpp
if (segmentCrossesHole(from_x, from_y, target.getX(), target.getY(),
                       hole_rings, 20)) {
    cost += 1000.0;  // ← 软惩罚：DP 仍可能选穿洞路径
}
```

per-cell veto 给部分 cell 分配不同于全局最优角的 swath 角度后，cell 间连接方向变化，DP 中所有 4×4=16 种 entry variant 组合都可能穿洞。+1000 软惩罚不足以保证 DP 避让。

### 修复：最小化改动

**commit**: `7e0a9af`

`path_planner.cpp` — `connectionCost` 穿洞惩罚从 `+1000.0` 改为 `+1e9`：

```cpp
// 硬约束：视穿洞连接为不可行
cost += 1e9;
```

**设计考量**：
- 不改动 per-cell veto 逻辑（保持 v9.9 的 `2.0x + 3` 离群阈值）
- 不增加迭代修复（老板的稳定妥协版没有，说明不需要）
- 1e9 是硬约束级别，确保 DP 在有可行连接时绝不选穿洞路径
- 若所有 variant 都穿洞，DP 选 +1e9 路径，后续由 `repairRouteConnectionsAroundHoles` 修复

### Per-cell 离群 Veto 代码（方向 B）

已通过 git diff/apply 应用到工作树分支。位于 `swath_generator.cpp:351-404`：

```
全局角度优化 → 选定 best_ang（全 cell 最少 swath 总数）
  ↓
逐 cell 检查：
  1. 计算 cell 本地最优角（computeCellMainDirection + 斜边检测）
  2. 在本地候选角度中选最少 swath 数
  3. 角度偏差 guard：本地角偏离全局 > 30° 时不 override
  4. 离群判断：全局 swath 数 > 本地最优 × 2.0 且多 ≥ 3 条
     → 是：应用本地角度（含 fillBoundaryGaps + filterShortSwaths）
```

### 经验教训

1. **软约束在组合优化中的危险性**：+1000 在大多数场景足够（cost 基数 ~数百），但在 all-variants-cross 的极端情况下 DP 无路可选，软约束等于没约束
2. **最小化修复原则**：老板的稳定妥协版无迭代修复、无收紧 veto 门槛 → 说明问题只在穿洞连接的约束硬度，不应过度工程化
3. **批测全覆盖验证**：单场景修复后必须跑全量批测确认零回归，避免"修一个问题引入另一个"
