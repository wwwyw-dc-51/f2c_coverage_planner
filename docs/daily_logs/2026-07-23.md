## 主要发现

### 1. 旧 S7 图片是失败批次残留

`batch_0723_0228` 中的 S7 图片来自失败运行，存在空路径和旧 planner 进程覆盖临时 JSON 的问题，不能作为当前算法结果判断依据。

### 2. 方形小回路的主要原因是 C-space 洞边界重复补线

机器人半宽为 `0.375 m`，车体尺寸已经纳入 C-space。部分狭窄区域中，已有 swath 已经覆盖了 C-space 洞边界，但 `fillBoundaryGaps` 再次沿洞边界补线，直接连接后形成窄矩形回路。

N3 修复前后对比：

- 路径点：368 → 261；
- 局部闭合小回路：61 → 27；
- 覆盖率保持 100%。

### 3. 连接图原先缺少真实路径数据

可视化 JSON 原先只保存 `from_cell/to_cell`，连接图无法绘制真实路线。现在连接项保存了 `route_waypoints` 路径，21 个场景共 1087 条连接全部包含有限坐标的 `path`。

## 当前仍需关注的问题

### 1. 原始覆盖率与修正覆盖率口径混用

当前报告同时存在原始网格覆盖率和扣除物理不可达空隙后的修正覆盖率。例如 S7 原始覆盖率约 98.60%，修正后为 100.00%。

批测解析脚本中的覆盖率正则没有严格锚定报告行，可能把“修正后覆盖率”误解析成 `eval.coverage_rate`。可视图又绘制原始红色未覆盖点，但角标使用修正值，因此会出现图上有红点、角标显示 100% 的现象。

后续应明确输出：

- `raw_coverage_rate`；
- `effective_coverage_rate`；
- `unreachable_area`；
- `reachable_uncovered_area`。

### 2. 路径效率仍可优化

- N3 重叠率 101.0%，综合得分 50.6；
- S7 重叠率 53.8%，路径总长 896.9 m，转弯 128 次；
- S3/N2 仍有少量可达边缘缺口。

下一步应优先研究共线边界 swath 合并和连接路径去冗余，不能直接恢复会导致穿洞的全局 normalize 或盲目边界补刀。

### 3. 审计脚本存在误报

`audit_test_report.py` 当前把 planner 的“有效转弯”与最终路径所有几何折点混为一谈，并且对精确位于 `0.375 m` 安全边界的路径存在浮点误报。扩展场景的 YAML 文件名映射也不完整，不能直接把其所有 ERROR 当作规划失败。
