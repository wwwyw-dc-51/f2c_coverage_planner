## 下午：Bug 修复 + 候选扫描 + 出界检测

### 修复的 Bug
1. **`cell_centers` 作用域错误** — 编译失败，声明在 `if` 块内但在块外引用
2. **`simplifyRing` 闭合环首顶点被跳过** — OGR LinearRing 首尾同点，`len1=0` 导致矩形第一个角被当作重复点删除，多边形严重变形
3. **边界检测只查中点** — 横跨房间的 swath 中点离墙 5m，阈值 0.9m 检测不到 → 边界 shrink 减半对 90% swath 失效。改为端点+中点三点检测
4. **`Swaths`/`SwathsByCells` 无 `isEmpty()` 方法** — 改为 `size()>0` / `sizeTotal()==0`
5. **`Swath` 无 `addPoint`/`addGeometry`** — Swath 是组合（含 LineString 成员），不是继承。需 `LineString::addPoint()` + `Swath(LineString)` 构造

### 新增功能
1. **候选 headland 扫描** — 9 组 headland 比 (0.30~0.12)，swath 层面快速评估覆盖率，选最优
2. **出界/碰障一票否决** — 路径 TURN 段采样检测，越界候选自动跳过换下一个
3. **多边形简化 `simplifyCells`** — 去除 OGR buffer 产生的共线冗余顶点，恢复规整多边形
4. **Cell 合并** — 邻接同向 cell 合并为一个大 cell，减少跨 cell 过渡（已关闭待验证）

### 删除的无效代码
1. **Cell 邻接对齐** — `std::reverse` 翻转 cell 内 swath 顺序，但 TSP 求解器把所有 swath 当独立节点优化，翻转无效
2. **`cell_is_boundary` 按 cell 索引** — `route.sizeVectorSwaths()` 返回 swath 总数（~100），不是 cell 数（~5），索引完全错位

### 尝试但回退的
1. **边界 margin=0** — 不收缩太激进，回退到 50%
2. **直连模式替代 Dubins** — 同 cell 内 swath 间直连斜穿场地，越界，回退到 PathPlanning+DubinsCurves
3. **headland=0.08** — 太小导致连接路径空间不够，回退到候选下限 0.12

### 脚本参数调整
- `mid_hl_width_ratio`: 0.25 → 0.15（候选扫描会自己试 0.30~0.12）
- `swath_endpoint_shrink_distance`: 0.25 → 0.08（差速驱动适配）

- 边界漏扫（headland gap ≈ 0.14m）
