## 模块职责

### 1. `polygon_planner_node.cpp` — 主节点 (~500行)
- ROS2 参数声明/获取
- 4 个多边形 Topic 订阅/发布
- `planCoveragePath` 调度流程（调用各模块）
- JSON 导出

**移入**: 不变，仅删除已移走的函数
**保留**: `updateParameters`, `publishEmptyPlanningOutputs`, `clearPlanningCacheForPolygon`, `createPolygonPath`, `createHolesOutlinePath`, `createDecompositionOutlinePath`

### 2. `decomposer.cpp/.hpp` — 区域分解 (~400行)
- `rectilinearDecompose` (Sweep 扫描线分解)
- `simplifyRing` / `simplifyCells`
- `computeCellMainDirection`
- `extractDecompositionAngles`
- `filterTinyCells`

### 3. `swath_generator.cpp/.hpp` — Swath 生成 (~600行)
- `optimizeSwathAngle` (多角度优化)
- `extractAllEdgeAngles`
- `detectSlantedBoundaryAngle`
- `swathLength` / `filterShortSwaths`
- `adjustSwathEndpoints` / `adjustSwathsEndpoints`
- `computePolygonMainDirection`
- `rotateCell` / `rotateSwath`

### 4. `boundary_filler.cpp/.hpp` — 边界补刀 (~500行)
- `fillBoundaryGaps` (外环 + 孔洞)
- v7 段级覆盖检测
- Cell 连接优化（反转 + 穿洞检测）

### 5. `path_planner.cpp/.hpp` — 路径规划 (~600行)
- genRoute 调用 + 路由拼接
- `simplifyPathRDP`
- v4 边界填充恢复
- 孔洞感知连接修复
- 掉头可行性检测

## 参数结构体

```cpp
struct DecomposerParams {
    bool use_sweep = true;
    double merge_angle_threshold = 60.0;
    bool filter_tiny = true;
    double min_cell_area_ratio = 2.0;
};

struct SwathParams {
    double coverage_width;
    double swath_overlap_ratio;
    bool angle_optimization;
    double min_swath_length;
    double endpoint_shrink;
};

struct FillParams {
    double coverage_width;
    std::string boundary_type;  // "closed"/"open"/"custom"
    double boundary_margin;
};

struct PathParams {
    std::string turn_planner_type;  // "direct"/"dubins"/"reeds_shepp"
    std::string swath_order_type;   // "boustrophedon"/"snake"
    bool simplify_enabled;
    double simplify_tolerance;
    std::string boundary_type;
    double endpoint_shrink;
    double boundary_margin;
};
```
