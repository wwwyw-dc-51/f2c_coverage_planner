### 2. rectilinearDecompose() sweep_only 模式 (~行 582-739)

**原逻辑**: 从 work_area 的所有顶点收集 x 和 y 坐标，排序去重后生成 `(xs.size()-1) × (ys.size()-1)` 个网格矩形。

**新逻辑**: 当 `sweep_only=true` 时：
- 只从孔洞顶点收集 y 坐标
- x 坐标只用外边界 min/max 两点 → 生成全宽水平条带
- 后处理：对贴近孔洞的 cell 做显式减法

**判定**: ✅ **优化**  
**风险**: 中  
**效能评估**:
- 孔洞场景 cell 数从 O(n²) 降到 O(n)（n=顶点数）
- 后处理减法有性能开销（每孔洞×每cell），但 cell 数大幅减少后总体更快
- BUG 风险：sweep_only 的条带可能与复杂多边形形状不匹配（窄通道、斜边）

### 3. fillBoundaryGaps() 改动 (~行 1346-1436)

**变更 A**: 移除了 `if (!touches_outer) return;` 的提前返回 → 即使 cell 不贴外环也继续处理孔洞边界  
**变更 B**: 移除了去重逻辑（原来检查新 swath 与已有 swath 的法向投影距离 < 0.3*cov_width 则跳过）→ 注释说"边界补刀不做去重"  
**变更 C**: swath 方向向量提前到外层计算（外环+孔洞共用）

**判定**: ⚠️ **正负混合**  

**正面效果**:
- 去除去重后能填补更多边界缝隙 → 覆盖率↑
- 移除提前返回让孔洞边界也能补刀

**负面风险**:
- 不区分外环/孔洞边界：孔洞边界补的 swath 是在"孔洞外侧"，方向判断可能出错
- 完全去除去重可能导致同一位置多条重叠 swath → 覆盖冗余、路径变长
- 原注释说"只跳过中心距过近(<0.05m)的真正重复"但实际代码中该过滤也删除了

**建议**: Phase 2 中恢复对孔洞边界的独立处理（2B），放宽而非完全去除角度容差

- 质心连线穿过孔洞 ≠ cell 不能合并（可通过孔洞一侧绕行）
