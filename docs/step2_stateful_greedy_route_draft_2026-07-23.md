# 第二步草稿：出口驱动的 Cell 联合贪心

状态：仅为下一步实现草稿，本轮不修改源代码。

## 目标

让上一个 Cell 的真实出口同时决定下一个 Cell、进入端、Cell 内部 Swath 方向和新的出口，避免先独立固定 Cell 状态后再被 F2C 路由器翻转。

## 计划

1. 保留现有 `greedyCellOrder` 的公开接口，但把候选选择抽成内部的 Cell 状态评估。
2. 每个未访问 Cell 生成合法入口变体，计算：
   - 当前出口到入口的安全连接距离；
   - Cell 内覆盖距离与转弯数；
   - 孔洞穿越硬约束；
   - 小矩形回路惩罚；
   - 后继 Cell 的一步前瞻代价。
3. 由贪心过程直接形成 Cell 块级顺序和每个 Cell 的 Swath 方向，不再让 OR-Tools 对全部 Swath 自由重排。
4. 用 Cell 级 2-opt 做局部交换；交换后重新选择相邻 Cell 的入口状态并重新验证安全性。
5. 保留现有 F2C `genRoute` 作为基线对照或失败回退，不把它和贪心结果串联覆盖。

## 必须保持的约束

- 不改变 Swath 覆盖几何，只改变顺序和方向；
- 不穿越孔洞、不离开规划区域；
- 空 Cell 保留原始索引映射；
- 旧路线与新路线都通过统一安全检查后才允许输出；
- 只有新路线在路径长度、连接长度或转弯数上改善且安全时才采用。

## 验收指标

- S7 跨 Cell 连接长度下降；
- Cell 间无意义跳转减少；
- N3 小矩形回路不增加；
- 覆盖率不下降；
- `hole_crossings=0`、`cspace_excursions=0`；
- 贪心结果不会再被默认 `redirect_swaths=true` 的 F2C 路由覆盖。
